Step 01

Colors &
Variables

Your fonts and colors are already loaded. Find your hex values in Figma's inspect panel and plug them into the background-color and color properties. Look for these lines in main.css and navigation.css — the comments tell you exactly where.

  background: rgb(12, 64, 64); ← swap this
}

/* where to change the background color of the pop-up nav */

Step 02

Type
Styling

Headings, body text, and navigation links each have their own CSS selectors. In Figma, inspect your type layers to find size, weight, color, and letter-spacing. Match those values to the corresponding selectors in main.css.

h2 {
  color: firebrick; ← your heading color
  font-variation-settings: "opsz" 16, "wght" 900, "slnt" -8;
}

nav a {
  color: rgb(163, 144, 144); ← nav link color
  font-size: 1.2rem;
}

Step 03

Grid
Layout

The desktop layout is controlled by grid-template-areas inside the @media query. Each named area maps to a section ID. Change the column ratios and area arrangement to match your mockup.

.page-wrapper {
  grid-template-columns: 1fr 1fr;
  grid-template-areas:
    "h v"
    "h a";
}

Step 04

Mobile
First

The layout shifts at 800px. Styles outside the media query apply to mobile; styles inside apply to desktop. If your mockup uses a different breakpoint, change min-width: 800px in both main.css and navigation.css.

/* mobile — no grid */
.page-wrapper { width: min(100% - 3rem, 95ch); }

/* desktop — grid kicks in */
@media only screen and (min-width: 800px) {
  .page-wrapper { display: grid; }
}

Step 05

Variable
Font Axes

Sour Gummy's axes are set via font-variation-settings. The wonk and width animation classes each animate between values. Adjust the start and end numbers in the @keyframes to match the expressiveness in your mockup.

SOUR
@keyframes weightChange {
  0%, 100% { font-variation-settings: "wght" 200, "slnt" 0; }
  50% { font-variation-settings: "wght" 900, "slnt" -13; }
}

Step 06

Timing &
Motion

Every animation has three knobs: duration (how long), easing (how it moves), and delay (when it starts). Tune these to match the energy of your design — a bold, chunky display might use slower, weightier timing than a clean minimal layout.
